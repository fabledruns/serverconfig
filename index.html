00
Prerequisites
What you need before running any of these commands.
01
Server Requirements
  • RAM — 4GB minimum, 8GB preferred
  • CPU — 2–4 cores
  • Disk — 30GB+ free
  • OS — Ubuntu Server 24.04 LTS
02
Find Your Server IP
You'll need this throughout the entire guide.
ip a
Note — Look for the inet address under ens3 or eth0. Write it down.
01
Install Docker
Use the official apt repo. Don't use snap or the default apt package — both are outdated and will cause issues.
01
Update the system
sudo apt update && sudo apt upgrade -y
02
Add Docker's official repo
sudo apt install -y ca-certificates curl gnupg

sudo install -m 0755 -d /etc/apt/keyrings

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo $VERSION_CODENAME) stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update

sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
03
Enable on boot
sudo systemctl enable docker
sudo systemctl start docker
04
Fix Docker DNS
sudo nano /etc/docker/daemon.json
Paste:
{
  "dns": ["1.1.1.1", "8.8.8.8"]
}
sudo systemctl restart docker
05
Optional — run Docker without sudo
sudo usermod -aG docker $USER
newgrp docker
Note — newgrp applies the change in the current session. Log out and back in to make it permanent.
02
Purpur Raw Docker
Run a Minecraft server with just Docker Compose — no panel. Worth understanding before moving to Pelican.
01
Clone the repo
git clone https://github.com/fabledruns/serverconfig.git root
cd root/minecraftserver
02
docker-compose.yml
services:
  mc:
    image: itzg/minecraft-server
    container_name: minecraft
    restart: unless-stopped
    environment:
      EULA: "TRUE"
      TYPE: "PURPUR"
      VERSION: "1.21.11"
      MEMORY: "4G"
      ONLINE_MODE: "false"   # disable Mojang auth for local play
    ports:
      - "25565:25565"
    volumes:
      - ./data:/data
03
Start and watch logs
docker compose up -d
docker logs -f minecraft
Wait for — "Done! For help, type "help"". Press Ctrl+C to exit the log tail.
04
Useful commands
docker compose stopstop the server
docker compose startstart it again
docker compose downstop + remove container (data stays)
docker exec -it minecraft rcon-cliinteractive server console
05
Key server.properties settings
nano data/server.properties
SettingWhat it does
view-distanceChunks sent to the client. Lower = less bandwidth.
simulation-distanceChunks that run game logic. Lower = less CPU.
max-playersPlayer cap.
motdServer description in the server list.
03
Pelican Panel
Pelican is an actively maintained Docker-native fork of Pterodactyl. The UI is nearly identical to what you've seen in most tutorials.
01
Navigate to Pelican directory
cd root/minecraftserver/pelican
cp .env.example .env
nano .env
02
Edit .env
APP_URL=http://YOUR_SERVER_IP     # no trailing slash
APP_TIMEZONE=Asia/Kolkata
DB_PASSWORD=your_strong_password
CACHE_DRIVER=redis
SESSION_DRIVER=redis
QUEUE_DRIVER=redis
03
Start the panel
docker compose up -d
Note — Wait 30–60 seconds before opening the URL. Containers need time to initialize.
04
Run migrations and create admin
docker compose exec panel php artisan migrate --force --seed
docker compose exec panel php artisan p:user:make
Set admin to yes when prompted.
05
Open the panel
http://YOUR_SERVER_IP
04
Wings Daemon
Wings is what actually runs game servers. It talks to the panel via API token and manages Docker containers on the host.
01
Install Wings binary
sudo mkdir -p /etc/pterodactyl

sudo curl -L https://github.com/pterodactyl/wings/releases/latest/download/wings_linux_amd64 \
  -o /usr/local/bin/wings

sudo chmod +x /usr/local/bin/wings
02
Create a node in the panel
  • Admin → Nodes → Create New
  • FQDN — your server's IP address
  • Scheme — http (no HTTPS needed)
  • Set RAM and Disk limits
  • Save → Configuration tab → copy the config.yml content
03
Save Wings config
sudo nano /etc/pterodactyl/config.yml
# paste the config from the panel
04
Create systemd service
sudo nano /etc/systemd/system/wings.service 
[Unit]
Description=Pterodactyl Wings Daemon
After=docker.service
Requires=docker.service

[Service]
User=root
WorkingDirectory=/etc/pterodactyl
LimitNOFILE=4096
ExecStart=/usr/local/bin/wings
Restart=on-failure
StartLimitInterval=180
StartLimitBurst=30
RestartSec=5s

[Install]
WantedBy=multi-user.target
05
Start Wings
sudo systemctl daemon-reload
sudo systemctl enable --now wings
sudo systemctl status wings
Expected — active (running). If not: sudo journalctl -u wings -f
06
Verify in panel
Admin → Nodes. Your node should show a green heartbeat. If it stays red after 30 seconds, check the FQDN in config.yml.
05
Create Server in Panel
With Wings connected, create a managed Purpur server through the panel UI.
01
Import the Purpur egg
Download the egg JSON from your browser:
https://github.com/parkervcp/eggs/blob/master/game_eggs/minecraft/java/purpur/egg-purpur.json
  • Admin → Nests → create a nest called "Minecraft"
  • Import Egg → select egg-purpur.json
02
Create an allocation
  • Admin → Nodes → your node → Allocations
  • IP: 0.0.0.0 — Port: 25566
03
Create the server
  • Admin → Servers → Create New
  • Assign to your node, pick the 25566 allocation
  • RAM — 4096 MB minimum
  • Nest: Minecraft → Egg: Purpur
  • Version — 1.21.11 in startup variables → Create Server
04
Install and start
  • Click Install — wait for Wings to download Purpur
  • Click Start → wait for "Done!" in the console tab
  • Connect: YOUR_SERVER_IP:25566

Optimizations
Tune JVM flags, spigot.yml, paper-world-defaults.yml, and purpur.yml. These settings compound — all of them together make a real difference.
01
JVM Flags — Aikar's Flags
Go to https://flags.sh, pick your RAM, copy the output. Paste into Admin → Servers → your server → Startup — replace the existing -Xms and -Xmx flags.
Why — Tunes G1GC to reduce lag spikes. Don't write these manually, just use flags.sh.
02
spigot.yml
world-settings:
  default:
    view-distance: 8         # chunks sent to client
    simulation-distance: 6  # chunks running game logic
    merge-radius:
      item: 2.5
      exp: 3.0
view vs simulation — view-distance is what the client sees. simulation-distance is what the server computes. Keeping simulation lower saves CPU without affecting visibility.
03
paper-world-defaults.yml
chunks:
  delay-chunk-unloads-by: 10s
  max-auto-save-chunks-per-tick: 24

mob-spawner-tick-rate: 2

entity-per-chunk-save-limit:
  experience_orb: 16
  arrow: 16
  fireball: 8
04
purpur.yml
settings:
  use-alternate-keepalive: true

world-settings:
  default:
    mobs:
      villager:
        brain-ticks: 2
        lobotomize:
          enabled: true
          check-interval: 100
      dolphin:
        disable-treasure-searching: true
lobotomize — Disables AI for idle villagers not near a bed or workstation. Doesn't affect trading. Big CPU win on larger servers.
05
Summary

simulation-distance6Fewer active chunks
